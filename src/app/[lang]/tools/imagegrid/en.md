# Online Image Grid Splitter for Instagram Grid and Social Media Tiles

This image grid splitter turns one image into a 3x3 grid, 3x1 panorama strip, or 3x2 poster wall. It is useful for Instagram Grid posts, Xiaohongshu image layouts, social media campaign visuals, poster splitting, and brand content planning. The tool builds a complete grid canvas locally in your browser, then slices it into square image tiles with preview, single download, and ZIP export.

## How to Split an Image into a Grid

1. Upload a JPG, PNG, WebP, or BMP image.
2. Choose a grid preset. The default preset is a 3x3 grid.
3. Set the tile size. 1080px is a common choice for social media.
4. Choose cover crop or contain padding. Cover crop fills the whole grid; contain padding keeps the full image and adds background color.
5. If cover crop is selected, adjust horizontal and vertical focus to keep faces, logos, or important text in view.
6. Generate the tiles and review each square image.
7. Download one tile or export all tiles as a ZIP file.

## Upload Order

Output files include an index and row/column marker, such as `grid_01_r1c1` and `grid_02_r1c2`. Upload order matters on social platforms. Instagram profiles often place the latest post first, so you may need to upload tiles in reverse order depending on the desired profile layout. For Xiaohongshu image posts, review the tile preview and choose files in the intended order.

## Cover Crop vs Contain Padding

Cover crop is best for posters, landscape photos, campaign visuals, and images that should fill every tile. It may crop edges to match the selected grid ratio.

Contain padding is best for product images, text-heavy posters, screenshots, and visuals where no content should be cropped. It keeps the full image and fills empty areas with the selected background color.

## Privacy and Image Quality

Image loading, preview rendering, tile slicing, and ZIP packaging all happen locally in your browser. Images are not uploaded to a server. The default 1080px tile size works well for most social platforms. If your source image is small, enlarged tiles may look soft; if your source image is very large, generating many high-resolution tiles may use more browser memory.
