# JSON 对比工具怎么用

这个在线 JSON 对比工具可以把两段 JSON 并排比较，并在规范化后高亮新增、删除和修改的内容。它适合对比 API 返回结果、配置文件、翻译字典、package.json、功能开关、请求参数和各种结构化 JSON 数据。

## 为什么规范化后的 JSON 对比更清楚

直接对比原始 JSON 很容易被缩进差异、键顺序变化、压缩格式这些噪音干扰。这个工具会先解析并校验左右两边的 JSON，再按统一格式整理后生成差异结果。开启键名排序后，对象键顺序变化不会盖过真正的数据变化。

## 常见使用场景

- 对比不同环境下的 API 响应 JSON。
- 检查接口请求参数在修改前后发生了哪些变化。
- 对比功能开关、应用配置、部署变量等 JSON 配置。
- 检查翻译 JSON、国际化字典和 locale 文件更新。
- 审查 package.json、manifest、测试夹具和 mock 数据修改。

## 校验、格式化、对比放在一起

左侧粘贴原始 JSON，右侧粘贴修改后的 JSON。每一侧都会显示 JSON 是否有效、包含多少个键，以及规范化后有多少行。如果 JSON 合法，还可以单独格式化左侧、右侧，或者一次把两边都整理成统一的可读格式，再导出差异报告。

## 浏览器本地处理

JSON 解析、格式化、对比和报告导出全部在浏览器本地完成，不上传服务器。无论是接口返回、配置片段还是内部调试数据，都可以直接在本机完成检查，适合调试、代码评审、发布前核对和日常排查。

## 导出 JSON 差异结果

当左右两边 JSON 都合法后，可以复制 JSON 差异报告，也可以下载 TXT 报告。报告会带上当前规范化设置、差异内容以及两边整理后的 JSON，便于留档、转发和写进变更记录。
